Methodology

How VULNRA publishes research

All research published here follows a strict sanitization protocol. No private endpoint data, customer information, or identifying deployment details are ever included. Findings are described in terms of probe technique, model family response, and aggregate statistics only.

When a new vulnerability pattern is discovered:

  1. Pattern is confirmed across a minimum of 3 independent deployments before publication
  2. Affected model providers are notified with a 30-day disclosure window
  3. Sanitized write-up is published with full OWASP / MITRE ATLAS mapping
  4. VULNRA's scan engine is updated to detect the new pattern automatically for all users

FAQ

Common questions

What is LLM vulnerability research?
LLM vulnerability research is the systematic discovery, documentation, and publication of security weaknesses in large language model deployments — including jailbreak techniques, prompt injection vectors, encoding bypasses, and data exfiltration paths. Unlike traditional software CVEs, LLM vulnerabilities are often behavioral and probabilistic rather than deterministic code bugs.
What is a jailbreak in the context of AI security?
A jailbreak is an adversarial input that causes a large language model to bypass its safety training and produce outputs it was explicitly trained to refuse — harmful content, private data, system prompt contents, or instructions for dangerous activities. Jailbreaks are typically probabilistic, meaning they succeed with a certain percentage of attempts rather than reliably 100% of the time.
What is prompt injection?
Prompt injection is an attack where malicious instructions embedded in user inputs or retrieved content override the intended behavior of an LLM application. Direct prompt injection targets the model's instruction-following behavior directly. Indirect prompt injection hides instructions inside documents, web pages, or database records that the model processes — making it especially dangerous in RAG and agent-based deployments.
What is OWASP LLM Top 10?
OWASP LLM Top 10 is a list of the ten most critical security risks in large language model applications, published by the Open Worldwide Application Security Project. The 2025 edition includes: LLM01 (Prompt Injection), LLM02 (Sensitive Information Disclosure), LLM03 (Supply Chain), LLM04 (Data and Model Poisoning), LLM05 (Improper Output Handling), LLM06 (Excessive Agency), LLM07 (System Prompt Leakage), LLM08 (Vector and Embedding Weaknesses), LLM09 (Misinformation), and LLM10 (Unbounded Consumption).
What is MITRE ATLAS?
MITRE ATLAS (Adversarial Threat Landscape for Artificial-Intelligence Systems) is a knowledge base of adversarial tactics, techniques, and case studies targeting AI systems — the AI-specific equivalent of MITRE ATT&CK. Key techniques relevant to LLM security include AML.T0051 (LLM Prompt Injection), AML.T0054 (LLM Jailbreak), AML.T0025 (Exfiltration via ML Inference API), and AML.T0043 (Craft Adversarial Data).
How does VULNRA's vulnerability research differ from academic AI safety research?
Academic AI safety research typically studies model behavior under controlled lab conditions using purpose-built test sets. VULNRA's research is empirical — derived from running adversarial probes against real production LLM deployments. This means findings reflect actual attack success rates in real-world configurations, including the system prompts, guardrails, and deployment contexts that real applications use, rather than base model behavior in isolation.
