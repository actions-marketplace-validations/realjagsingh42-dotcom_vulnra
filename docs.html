API Documentation

The VULNRA API lets you programmatically scan LLM endpoints for jailbreaks, prompt injections, encoding bypasses, and compliance violations. Integrate security testing into CI/CD pipelines, automate compliance reporting, or build custom dashboards.

Quick Start

Make your first API call in two steps: get an API key from your dashboard, then POST to /scan.

cURL
Python
JavaScript
# Start a scan
curl -X POST "https://vulnra-production.up.railway.app/scan" \
  -H "X-API-Key: vk_your_api_key" \
  -H "Content-Type: application/json" \
  -d '{"url":"https://api.example.com/chat","tier":"pro"}'
import requests

API_KEY = "vk_your_api_key"
BASE_URL = "https://vulnra-production.up.railway.app"

# Start a scan
response = requests.post(
    f"{BASE_URL}/scan",
    headers={"X-API-Key": API_KEY},
    params={"url": "https://api.example.com/chat", "tier": "pro"}
)

result = response.json()
print(f"Risk Score: {result['risk_score']}")
print(f"Findings: {len(result['findings'])}")
const scan = await fetch(
  `https://vulnra-production.up.railway.app/scan?url=${encodeURIComponent(targetUrl)}&tier=pro`,
  { headers: { 'X-API-Key': 'vk_your_api_key' }, method: 'POST' }
);

const result = await scan.json();
console.log(`Risk Score: ${result.risk_score}`);
console.log(`Findings: ${result.findings.length}`);

A successful scan returns the full result object immediately (synchronous mode). See the POST /scan reference for all parameters.

Authentication

All API requests require an API key passed in the X-API-Key header. Get your key from the scanner dashboard under Settings → API Keys.

Request Header
X-API-Key: vk_xxxxxxxxxxxxxxxxxxxx
Security: Never expose your API key in client-side code or public repositories. Use environment variables. Rotate keys immediately if compromised via the dashboard.

Key format

All API keys follow the format vk_ followed by 20 alphanumeric characters. Keys are scoped to your account tier and enforce the rate limits of your plan.

Base URL & Headers

Production https://vulnra-production.up.railway.app

All endpoints accept and return application/json except /report/generate which returns application/pdf.

Required Headers
X-API-Key: vk_your_api_key
Content-Type: application/json

Endpoints

POST /scan Start a security scan ▾

Initiates a security scan against the target LLM endpoint. Returns the full result immediately in synchronous mode, or a scan_id in queued mode when Celery is available. All findings are tier-gated — free tier returns 1 blurred finding, Pro returns all.

Query Parameters
Name Type Required Description
url string required Full URL of the target LLM REST API endpoint. Must be HTTPS. RFC 1918 ranges and localhost are blocked.
tier string optional Scan tier: free | pro | enterprise. Defaults to your account tier.
strategy string optional single_turn (default) or multi_turn. Multi-turn available on Pro+.
scan_type string optional endpoint (default) | mcp. MCP scanner in beta.
Response — 200 OK
200 OK application/json

GET /scan/{scan_id} Poll scan status or retrieve completed result ▾

Returns the current status of a scan. When status === "complete" the full findings object is included. When running in synchronous mode, the POST /scan response already contains the full result — polling is only needed in Celery-queued mode.

Path Parameters
Name Type Required Description
scan_id string required The scan_id returned from POST /scan.
Polling tip: If you're using async mode, poll every 3 seconds. Most scans complete in under 60 seconds. Abandon polling after 120 seconds and re-submit.
POST /report/generate Generate a compliance-mapped PDF audit report ▾

Accepts the full scan result JSON and returns a formatted compliance audit PDF. The report includes risk score, all findings with severity, framework citations (EU AI Act, DPDP, NIST, OWASP), and an executive summary section. Available on Pro and Enterprise tiers.

Request Body (application/json)
Name Type Required Description
scan_result object required Full scan result object as returned by POST /scan or GET /scan/{id}.
Response
200 OK application/pdf
# Binary PDF stream. Save with Content-Disposition header.
Content-Disposition: attachment; filename="vulnra-audit-sc_9f2a1b3c4d.pdf"
Content-Type: application/pdf

Sentinel Monitoring

Sentinel lets you schedule recurring scans on a cron expression. VULNRA re-runs the probe suite on schedule and alerts you via Slack or email when new vulnerabilities appear or the risk score increases by more than 1.0 point.

Availability: Sentinel is available on Pro and Enterprise tiers. Coming Q2 2026 — subscribe to the changelog for launch notification.
POST /sentinel/create Create a scheduled monitoring watch ▾
Request Body
Name Type Required Description
url string required Target LLM endpoint URL to monitor.
cron string required Standard cron expression. Example: 0 9 * * 1 for weekly Monday 9am UTC.
alert_webhook string optional Slack incoming webhook URL for alert delivery.
alert_email string optional Email address for alert delivery. Defaults to account email.
drift_threshold number optional Alert when risk score increases by this amount. Default: 1.0.

Rate Limits

Rate limits are enforced per API key. Exceeding the limit returns a 429 Too Many Requests response with a Retry-After header.

Tier Scans / Day API Requests / Min PDF Reports / Day
Free 1 10 1
Pro 100 60 100
Enterprise Unlimited 600 Unlimited

Error Codes

All errors return a JSON object with error and detail fields.

Error Shape
{ "error": "PRIVATE_IP_BLOCKED", "detail": "Private IP ranges are not permitted as scan targets." }
Status Error Code Description
400 PRIVATE_IP_BLOCKED Target URL resolves to a private IP range, localhost, or cloud metadata endpoint.
400 INVALID_URL The target URL is malformed or not HTTPS.
401 INVALID_API_KEY API key is missing, malformed, or revoked.
403 TIER_INSUFFICIENT The requested feature (e.g. multi-turn) requires a higher tier.
429 RATE_LIMIT_EXCEEDED Daily or per-minute scan limit reached. Check Retry-After header.
500 SCAN_ENGINE_ERROR Internal error in the scan engine. Retry after 30 seconds. If persistent, contact support.

GitHub Action

Add LLM security scanning to any repository. The action calls the VULNRA API and fails the build if the risk score exceeds your threshold. Once in your CI pipeline, every model deployment is automatically tested.

.github/workflows/ai-security.yml
name: VULNRA LLM Security Scan

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  security-scan:
    runs-on: ubuntu-latest
    steps:
      - name: VULNRA Security Scan
        uses: vulnra/vulnra-action@v1
        with:
          target_url: ${{ secrets.LLM_ENDPOINT }}
          api_key: ${{ secrets.VULNRA_API_KEY }}
          fail_threshold: 7.0
          tier: pro

Add VULNRA_API_KEY and LLM_ENDPOINT to your repository's Settings → Secrets. The action exits with code 1 if risk_score > fail_threshold, blocking the merge.
