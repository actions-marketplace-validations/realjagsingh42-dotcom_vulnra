v0.8.0
February 14, 2026
PDF Reports + Sync Scan Engine
Compliance PDF generation fully working end-to-end. POST /report/generate now accepts full scan JSON directly — no Redis dependency. Scans return synchronously on Railway free tier.
New PDF audit reports via ReportLab 4.1.0. Generates compliance-mapped output with findings, risk scores, EU AI Act / DPDP / NIST citations, and remediation notes.
New Synchronous scan mode — POST /scan falls back to synchronous _run_scan() when Celery is unavailable. Returns full result immediately.
Improved POST /report/generate now accepts the full scan JSON in the request body. No Redis lookup required — works even when Redis is unavailable on Railway.
Fixed PDF download broken — reportlab was missing from requirements.txt and scan data was lost between requests. Both resolved.
Fixed Scan results lost on Railway — results stored in _memory_store were stateless per-request. Sync mode eliminates the dependency entirely.
v0.7.0
January 22, 2026
Railway Deployment + Redis Fix
Production backend fully deployed on Railway with auto-deploy from git. Critical Redis environment variable ordering bug fixed. Garak mock fallback implemented for Railway's memory constraints.
New Railway auto-deploy — backend deploys on every git push origin main. Netlify handles frontend. Zero manual deployment steps.
New Garak mock fallback — when Garak is unavailable on Railway (no torch/transformers), garak_engine.py returns realistic mock findings with tiered blurring by subscription tier.
Fixed Redis not connecting on Railway — _env() in worker.py was reading .env before os.environ, so the Railway REDIS_URL was ignored. Reordered to check os.environ first in both files.
Fixed "Failed to fetch" errors — frontend cached old localhost:8000 URL. Railway served a stale build. Fixed by deleting runtime.txt and force-redeploying Netlify.
Security GET /debug/redis endpoint added — shows connection status and URL prefix for diagnosis. Returns connected / unavailable status.
v0.6.0
December 10, 2025
Compliance Mapping — EU AI Act + India DPDP
Full compliance framework mapping implemented. Every scan finding now maps to specific EU AI Act articles, India DPDP sections, NIST AI RMF functions, and OWASP LLM Top 10 categories.
Differentiation

India DPDP mapping is unique to VULNRA in the self-serve LLM security category. No other tool maps LLM vulnerabilities to India's Digital Personal Data Protection Act. Section 8, 11, and 16 citations are now included in every Pro scan.

New EU AI Act mapping — Article 9 (risk management), Article 13 (transparency), Article 15 (accuracy), Article 17 (quality). Fine exposure shown: up to €30M.
New India DPDP mapping — Section 8 (data principal obligations), Section 11 (consent), Section 16 (cross-border transfer). Fine exposure: up to ₹250Cr. First in category.
New NIST AI RMF mapping — GOVERN 1.1, MAP 2.1, MEASURE 2.5, MANAGE 1.1. Full four-function coverage.
New ISO 42001 mapping — Clause 6.1, Clause 9.1, Annex A controls. Compliance accordion in scanner results panel.
Improved Compliance blurring for Free tier — framework citations locked behind Pro. Users can see that compliance issues exist but not which articles are implicated.
v0.5.0
November 3, 2025
Garak Integration + Risk Scoring
NVIDIA Garak v0.14.0 integrated as the primary scan engine. Deterministic 0–10 risk scoring live. Eight vulnerability categories implemented including jailbreak, prompt injection, encoding bypass, and PII exfiltration.
New Garak v0.14.0 integration — full probe library including AutoDAN, DAN variants, indirect injection chains, encoding permutations (Base64, ROT13, Unicode homoglyphs).
New 0–10 risk score — deterministic scoring weighted by severity, exploitability, and regulatory exposure. Comparable across scans over time.
New Tiered finding blurring — Free tier shows 1 blurred finding. Pro shows all findings in full. Blurring is server-side — cannot be bypassed from the frontend.
Improved Async Celery queue — scans run as Celery tasks with Redis broker. GET /scan/{scan_id} polls status. Falls back to synchronous if Celery is unavailable.
