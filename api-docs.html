VULNRA API Reference

Red-team any LLM endpoint programmatically. One POST request returns a complete security audit — risk score, vulnerability findings, and compliance mapping across EU AI Act, NIST AI RMF, OWASP LLM Top 10, and India DPDP.

Base URL: https://api.vulnra.ai/v1
All requests require Bearer token authentication. Free tier: 1 scan/day. Pro tier: 100 scans/day. Enterprise: unlimited.

Authentication

All VULNRA API requests require an API key passed as a Bearer token in the Authorization header.

Authorization: Bearer YOUR_API_KEY

Quick Start — curl

curl -X POST https://api.vulnra.ai/v1/scan \
  -H 'Authorization: Bearer YOUR_API_KEY' \
  -H 'Content-Type: application/json' \
  -d '{"target_url": "https://your-llm-endpoint.com/chat", "tier": "pro"}'

Endpoints

Method Endpoint Description Auth
POST /v1/scan Start a new scan. Returns risk score, findings, and compliance map. Yes
GET /v1/scan/{scan_id} Poll scan status. Returns full result when complete. Yes
POST /v1/report/generate Generate compliance PDF from scan JSON. Returns application/pdf. Yes
GET /v1/scans List all previous scans for the authenticated user. Yes
GET /v1/health Returns system status, scan mode, and Redis connection state. No

FAQ

How do I authenticate with the VULNRA API?
Pass your API key as a Bearer token in the Authorization header of every request. Format: Authorization: Bearer YOUR_API_KEY.
What LLM endpoints does VULNRA support?
VULNRA works with any LLM REST API that accepts a JSON POST request with a messages array and returns a text response.
