GitHub Actions
CI / CD
Coming Soon

Fail your build automatically if your LLM endpoint's risk score exceeds a threshold. One YAML file. No infrastructure. Every push to main triggers a security scan before deployment.

1Add VULNRA_API_KEY to your repo secrets
2Copy the workflow YAML into .github/workflows/
3Set your fail_threshold (default: 7.0)
# .github/workflows/ai-security.yml name: VULNRA Scan on: [push] jobs: scan: runs-on: ubuntu-latest steps: - uses: vulnra/vulnra-action@v1 with: target_url: ${{ secrets.LLM_ENDPOINT }} api_key: ${{ secrets.VULNRA_API_KEY }} fail_threshold: 7.0

Same security coverage for GitLab pipelines. Add VULNRA as a CI job that runs against your staging or preview LLM endpoint before merging to main. Supports merge request blocking.

# .gitlab-ci.yml vulnra-security: stage: test image: python:3.11-slim script: - pip install vulnra-cli - vulnra scan $LLM_ENDPOINT --fail-above 7.0 allow_failure: false

Receive instant Sentinel alerts in your Slack channel when a new vulnerability appears in your model or when the risk score rises. Configurable per watch — send critical alerts to #security and summaries to #engineering.

1Create a Slack Incoming Webhook in your workspace
2Add the webhook URL when creating a Sentinel watch
3Set your drift_threshold (default: 1.0 point)
# Sentinel watch with Slack alert POST /sentinel/create { "url": "https://api.example.com/chat", "cron": "0 9 * * 1", "alert_webhook": "https://hooks.slack.com/...", "drift_threshold": 1.0 }

REST API

Build any integration yourself

If we don't have a native integration for your tool, the VULNRA REST API gives you full programmatic access to start scans, retrieve results, generate reports, and manage Sentinel watches.

POST /scan — start a scan synchronously or async
GET /scan/{id} — poll status or retrieve full results
POST /report/generate — get a compliance PDF
POST /sentinel/create — schedule recurring monitoring
API keys scoped to tier, rotatable from dashboard
Rate limits: Free 1/day · Pro 100/day · Enterprise unlimited
# Full scan + PDF in one script import requests API = "https://vulnra-production.up.railway.app" KEY = "vk_your_api_key" HDR = {"X-API-Key": KEY} # 1. Start scan res = requests.post( f"{API}/scan", headers=HDR, params={"url": "https://your-llm.com/v1/chat", "tier": "pro"} ).json() print(f"Risk: {res['risk_score']} | Findings: {len(res['findings'])}") # 2. Download PDF pdf = requests.post( f"{API}/report/generate", headers=HDR, json={"scan_result": res} ) open("audit.pdf", "wb").write(pdf.content)

Webhooks

Trigger scans from any deploy event

POST to the VULNRA Sentinel webhook endpoint from any deployment system — Vercel, AWS CodeDeploy, Kubernetes, custom scripts. Every deploy triggers an immediate security scan.

The webhook accepts a watch_id and an optional commit_sha for the changelog. Results are posted to your configured Slack channel and stored in the dashboard.

Webhook docs →
# Trigger on deploy (curl from any CI) POST https://vulnra-production.up.railway.app/sentinel/webhook { "watch_id": "wt_a1b2c3d4e5", "commit_sha": "abc123", "deploy_env": "production", "triggered_by": "vercel-deploy-hook" } # Response — scan queued { "scan_id": "sc_9f2a1b3c4d", "status": "queued", "eta_seconds": 45 }
